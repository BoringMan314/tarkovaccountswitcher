## Project Overview

Tarkov Account Switcher is a Windows-only desktop app for managing multiple Escape from Tarkov accounts. It automates BSG Launcher session-token swapping so users can switch accounts with one click (no passwords stored).

The active codebase is the Go + Wails v2 rewrite under `v2/`. The root README.md still mentions `lxn/walk`, but that stack has been replaced — **all real work happens inside `v2/`**.

## Build & Run

All commands run from `v2/`:

```bash
wails dev                                  # hot-reload dev mode
wails build -platform windows/amd64        # release build → v2/build/bin/Tarkov Account Switcher.exe
go mod tidy                                # resolve dependencies
go run sync_version.go                     # sync productVersion in wails.json from updater.go (run before a release build)
```

No test or lint scripts. No frontend build step — `frontend/dist/` is vanilla HTML/CSS/JS, served directly.

**Build prerequisites:** Go 1.23+ (toolchain pinned to 1.26.2), Wails CLI v2 (`go install github.com/wailsapp/wails/v2/cmd/wails@latest`), and MinGW-w64 GCC in PATH (Wails needs CGO).

## Versioning

`CurrentVersion` in `internal/updater/updater.go` is the single source of truth. `sync_version.go` (build-tagged `//go:build ignore`) rewrites `wails.json`'s `productVersion` to match. Run it before release builds so the PE resource version matches the updater check.

## Architecture

```
main.go        Wails entry: embeds frontend/dist, configures window + SingleInstanceLock, binds App
app.go        Go↔JS bridge. All PUBLIC methods on *App are auto-exposed to the frontend
              as window.go.main.App.*. Returns DTOs (never raw session data).
tray_windows.go  Native Win32 Shell_NotifyIconW tray on its own goroutine (no tray library).
                 Also sets the window icon via Win32 (reliable regardless of build path).

internal/
  accounts/   CRUD, AES-256-CBC (random IV, PKCS7), 2s polling watcher w/ 5min timeout.
              SessionCapturedCallback fires on capture → emits "session-captured" event to JS.
  launcher/   taskkill/start BSG launcher, settings file read/write, cache clearing.
              OnLauncherStarted callback → app hides window to tray.
  config/     Settings JSON, %APPDATA% paths, email masking (streamer mode), autostart registry key.
  i18n/       DE/EN translations. Language chosen from settings or system locale.
  updater/    GitHub Releases poller. CurrentVersion constant. Emits "update-available" event.
  singleinstance/  Windows mutex (fallback beside Wails' built-in SingleInstanceLock).
```

**Go → JS communication:** `runtime.EventsEmit(ctx, "session-captured" | "update-available", ...)`. The frontend (`frontend/dist/app.js`) listens via the Wails JS runtime in `frontend/wailsjs/runtime`.

**JS → Go:** auto-generated bindings under `frontend/wailsjs/go/` — regenerated by `wails dev`/`wails build` from the public methods on `*App`. Don't hand-edit them.

**DTO boundary:** `app.go` defines `AccountDTO`, `SwitchResultDTO`, `SettingsDTO`. Emails are masked through `config.MaskEmail` before crossing to JS when streamer mode is on. Raw session tokens never leave Go.

## Data & Platform

- Storage: `%APPDATA%\TarkovAccountSwitcher\` — `accounts.json` (encrypted), `settings.json`, `.key` (AES-256 key, mode 0600).
- Launcher settings read/written at `%APPDATA%\Battlestate Games\BsgLauncher\settings`.
- Windows-only: `taskkill`, Windows Mutex, Win32 tray, autostart via registry. Don't add cross-platform abstractions unless asked.
- `HideWindowOnClose: true` — closing the window minimizes to tray; use `QuitApp()` / tray "Quit" to actually exit.
- No code signing → SmartScreen warnings on install are expected.

## Frontend Notes

- Single page, 3 tabs (Accounts / Add / Settings) in `frontend/dist/index.html`.
- Theming via `[data-theme="..."]` CSS custom properties in `style.css` (5 themes: eft, killa, dark, light, cappuccino). Adding a theme = new CSS block + `<option>` in the dropdown.
- No npm, no bundler, no framework. Keep it that way unless the user asks to change it.
